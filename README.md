## `$ cat architecture.md`

```
HOW I BUILD SYSTEMS
═══════════════════

MOBILE (Flutter)
─────────────────
┌──────────────────────────────────────────────────────┐
│  UI Layer          → Widgets / Screens / Components  │
│  State Layer       → Provider / ChangeNotifier       │
│  Repository Layer  → Abstract data contracts         │
│  Service Layer     → API calls / local storage       │
│  Data Layer        → Models / JSON serialization     │
└──────────────────────────────────────────────────────┘
  Rule: One file = One responsibility.
  Rule: No business logic inside Widget tree.
  Rule: Provider handles state. Widget just renders.
  Rule: SharedPreferences for persistence, not hacks.

WEB (Laravel)
─────────────
┌──────────────────────────────────────────────────────┐
│  Routes      → Clean RESTful API design              │
│  Controllers → Thin. Logic lives in Services.        │
│  Services    → Business logic isolated & testable    │
│  Models      → Eloquent. Relationships well-defined  │
│  Views       → Blade / JSON responses only           │
└──────────────────────────────────────────────────────┘
  Rule: Fat models are fine. Fat controllers are sins.
  Rule: Validation in Form Requests, not Controllers.
  Rule: Every route has a purpose. No dead routes.

GENERAL PHILOSOPHY
──────────────────
  > Code is read more than it is written. Write clearly.
  > If a function does 2 things, it's 2 functions.
  > Name things what they are, not what you want them to be.
  > Commit small. Ship often. Fix fast.
  > A working prototype beats a perfect plan.
```
